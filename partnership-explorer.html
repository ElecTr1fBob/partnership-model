Mathematical Specification

Complete formal definition of the partnership equity function for verification and analysis

1. Core Formula — The Piecewise Function

Daniel’s equity share D(x) as a function of exit value x (in millions USD):

D(x) = { 0.50 if x ≤ 6.8 0.50 + 0.30 · f(x) if 6.8 < x < 110 0.80 if x ≥ 110 }

where:

f(x) = [1 − e^(−k · (x − x₀)^p)] / [1 − e^(−k · (x_max − x₀)^p)]

with parameters:

ParameterValueMeaning
x₀6.8Transition point — exit value where equity shift begins ($6.8M)
x_max110Saturation point — exit value where Daniel reaches maximum equity ($110M)
k0.05Rate parameter — controls steepness of the S-curve transition
p0.40Power parameter — creates concavity (sublinear growth via power-law input)
D_min0.50 (50%)Daniel’s minimum equity share (equal split baseline)
D_max0.80 (80%)Daniel’s maximum equity share (cap)

2. Function Design Rationale

2.1 Why a Modified Complementary Exponential CDF?

The core building block 1 − e^(−t) is the CDF of an exponential distribution. It maps [0, ∞) → [0, 1) with natural saturation behavior — fast initial growth that gradually levels off.

2.2 Why the Power-Law Input Transform?

Instead of using the raw distance (x − x₀), we apply (x − x₀)^0.40. Since p = 0.40 < 1, this is a concave transform that compresses large values more than small ones, creating diminishing returns.

2.3 Why the Normalization Denominator?

The denominator ensures f(x_max) = 1 exactly. Without it, the exponential CDF approaches 1 asymptotically but never reaches it.

Denominator = 1 − e^(−0.05 · 103.2^0.40) ≈ 1 − e^(−0.31955) ≈ 1 − 0.72634 ≈ 0.27366

2.4 The Complete Expansion

For 6.8 < x < 110: D(x) = 0.50 + 0.30 · [1 − e^(−0.05 · (x−6.8)^0.40)] / 0.27366 Or equivalently: D(x) = 0.50 + 1.0963 · [1 − e^(−0.05 · (x−6.8)^0.40)]

3. Key Mathematical Properties

Continuity at x = 6.8:
lim(x→6.8⁺) D(x) = 0.50 + 0.30 · f(6.8) = 0.50 + 0.30 · 0 = 0.50 = D(6.8) ✓
Continuity at x = 110:
lim(x→110⁻) D(x) = 0.50 + 0.30 · f(110) = 0.50 + 0.30 · 1 = 0.80 = D(110) ✓
Monotonicity (D is non-decreasing):
For x ≤ 6.8: constant ✓
For x ≥ 110: constant ✓
For 6.8 < x < 110: D'(x) = 0.30 · f'(x) > 0 (composition of increasing functions) ✓
Bounded range: D(x) ∈ [0.50, 0.80], Lior's share L(x) = 1 − D(x) ∈ [0.20, 0.50] ✓

7. Asymptotic Behavior

As x approaches infinity (beyond the model range): Without capping: D(x) approaches 0.50 + 1.0963 = 1.5963 This exceeds 1.0, which is why the cap at D_max = 0.80 is essential. The cap engages at x = 110M where f(x) = 1 exactly.

8. Design Principles Summary

Equal baseline: At modest exits (up to $6.8M), partners split 50/50.

Performance reward: Beyond $6.8M, Daniel's operational execution earns additional equity.

Diminishing marginal shift: The concave power transform (p=0.40) ensures each additional million shifts less equity than the previous one.

Hard cap protection: Lior never drops below 20%, preserving meaningful co-ownership.

Mathematical elegance: A single continuous, monotone, bounded function with no discontinuities, no edge cases, fully deterministic.
